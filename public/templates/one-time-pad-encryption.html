The One-time pad is a simple encryption algorithm that is the most secure one, as long as truly random data is possible to generate.

The idea of this encryption method is the following: suppose we have the plain text and want to encrypt it. We can generate a random key of same size of this plain text string, then match every binary representation of each character of both key and text to generate a ciphertext. The XOR is an operator that takes as input two bits (0 or 1) and returns one bit (0 or 1), depending on the inputs. If the inputs are equal, then it returns 0. If they are different, it outputs 1.

$$ 0 \oplus 0 = 0 $$ $$ 1 \oplus 0 = 1 $$ $$ 0 \oplus 1 = 1 $$ $$ 1 \oplus 1 = 0 $$

Given the output, we cannot crack the ciphertext without knowing both plain text and key inputs, which makes this encryption pretty strong. The problem is that to encrypt a whole string, we have to do it bitwise since this algorithm only works for bits. Without further ado, let's dive into the code.

One last thing needs to be done. You see, when we perform the bitwise XOR operator using the plain text and the key, we can end up with a 8-bit binary sequence that is represented by an "ilegal" ASCII character, such as "\t" (tab), "\n" (new line) and so on. This kind of pollutes our ciphertext. To remove this annoyance, we simply add another method that makes sure our ciphertext has only the allowed ASCII characters. This method basically converts every 8-bit binary sequence representing a ciphertext character into decimal. If this decimal number is in between 33 and 126 (allowed characters), we let it as it is. If not, then we replace it by the null character. This does not destroy our encryption because null ASCII characters are still countable in length such that the ciphertext, even though possibly shorter then the plain text to our eyes, will still be convertable to its original binary format. If we have the binary representation of the key and the ciphertext, we can decrypt the plain text still.
