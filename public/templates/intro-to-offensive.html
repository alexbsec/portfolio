Web Application Security

A web application is like a “program” that we can use without installation as long as we have a modern standard web browser, such as Firefox, Safari, or Chrome. Consequently, instead of installing every program you need, you only need to browse the related page. The following are some examples of web applications:

  • Webmails: Gmail, Outlook, Yahoo mail.
  • Online Shopping: Amazon, Etsy, AliExpress.
  • Social Media: Facebook, Instagram, Snapchat.

Consider an online shopping application. Usually, these kind of applications read data about the products and their details from a database server. A database server is responsible for many functions, including reading, searching, and writing to the database. The online shopping web application might need more than one database to access, for example:

  • Products database: This database contains details about the products, such as name, price, stock, specifications etc.
  • Customers database: This database contains details about the users/customers, such as name, email, address, credit card etc.
  • Sales database: We expect to see what each customer has purchased and how they paid in this database.

Web Application Security Risks

Suppose we want to buy an item from an online shop. These are some functions one should expect to be able to do within this web app:

  • Log in on the website
  • Search for the product
  • Add the product to cart
  • Specify the shipping address
  • Provide payment details

There may be a few common vulnerabilities associated with each one of these functionalities:

  • Log in at the website: The attacker can try to discover the password by brute forcing; or the attacker could use a more robust technique such as SQL injection to retrieve users credentials.
  • Search for a product: The attacker can attempt to breach the system by adding specific characters and code injection to the search term.
  • Provide payment details: The attacker would check if the payment details are sent in cleartext or using weak encryption. Encryption refers to making the data unreadable without knowing the secret key or password.

Identification and Authentication Failure

Identification refers to the ability to identify a user uniquely. In contrast, authentication refers to the ability to prove that the user is whom they claim to be. The online shop must confirm the user’s identity and authenticate them before they can use the system. However, this step is prone to different types of weaknesses. Example weaknesses include:

  • Allowing the attacker to brute force the login field, usually using automated tools, to find valid login credentials.
  • Allowing the user to choose a weak password. This can be combined with the above method.
  • Storing the user's credentials in plain text. If the attacker manages to find the file containing the user's credentials, he can easily retrieve and learn the user's password.

Broken Access Controls

Access control ensures that each user can only access files (documents, files, etc.) related to their role in the web application. For example, you don't want someone in the marketing department to access (read) the finance department's documents and/or files. Examples of vulnerabilities of access control include:

  • Failing to apply the principle of least privilege and giving users more access permissions than they actually need. For example, an online customer should be able to view the prices of the items, but they should not be able to change them.
  • Being able to view or modify someone else's account by using its unique identifier (uid). For example, you don’t want one bank client to be able to view the transactions of another client.
  • Being able to browse pages that require authentication as an unauthenticated user. For example, we cannot let anyone view the webmail before logging in.

Injection

Injection is a type of vulnerability where an attacker injects malicious code into some of the web application's functionality. This can have several implications, from retrieving information from a database that otherwise were not supposed to be retrieved; inject HTML or Javascript code to deface webpages and create malicious links; inject malicious code in XML files to make the backend server to behave in a unexpected manner.

Cryptographic Failures

Cryptography is the process of encrypting and decrypting data. The encryption part transforms plain text into ciphertext, whereas the decryption focuses in the inverse process. In order to encrypt something, a key is generated. This key can be used to either decrypt the data (when the encryption is not one-way only), but also to encrypt that plain text to the desired ciphertext. Examples of cryptographic failures include:

  • Sending sensitive data in a clear text. For example, using HTTP rather than HTTPS.
  • Relying on a weak and predictable encryption algorithm. One old cryptographic algorithm is to shift each letter by one.
  • Using weak keys for cryptographic functions. It won’t be challenging to break the encryption that used '1234' as a key.

Introduction to Operating System Security

To define an operating system, we need to visit one computer term: hardware.

Computer hardware refers to all the computer parts and peripherals that you can touch with your hands. It includes the screen, mouse, keyboard, printer, USB flash memory, RAM memory, SSD, HDD, mother board and so on.

The mother board is the main part of the computer, and all the other pieces of hardware end up needing to connect to the mother board. However, hardware components by themselves are useless if you want to run your favorite programs and applications. We need an Operating System to control and “drive” them.

Operating Systems

The operating system (OS) serves as an intermediary layer between hardware and software, facilitating seamless user interaction with software while also ensuring proper hardware utilization. Softwares are the essence of everything we do inside the computer. Unlike hardware, software refers to the applications and programs executed by the computer. While these programs rely on a proper hardware configuration, they also require an OS to accurately interpret and execute them.

Securing an Operating System

Every OS stores sensitive and private data that must be protected. When we talk about security, we should think of protecting three main things:

  • Confidentiality: One wants to ensure that secret and private data are only available to intended persons.
  • Integrity: It is crucial that no one can tamper with the files stored in your system or while being transferred on the network.
  • Availability: One wants their system to be available to retrieve its data anytime needed.

This is called the CIA triad. Usually, when an attacker wants to invade a system, it aims to break at least one of these three categories. Let's see the most common type of attacks that happens on OS.

1. Authentication and Weak Passwords

As already mentioned, authentication is the act of verifying one's identity, and it can be achieved via three main ways:

  • Something you know, such as password or PIN.
  • Something you have, such as a phone number or email.
  • Something you are, such as a fingerprint.

Usually, every OS has ways to implement at least one of these three authentication methods. Since passwords are the most common form of authenticate someone, they are also the most attacked method. Unfortunately, many users opt for easily guessable passwords or reuse the same password across multiple websites and platforms. In essence, if an attacker manages to deduce the password for any of your online accounts, such as your email or social media, they gain unrestricted access to your private information. Hence, it is of utmost importance to select intricate passwords and use distinct passwords for various accounts.

2. Weak File Permissions

Proper security dictates the principle of least privilege. In a work environment, you want any file accessible only by those who need to access it (ensuring availability). On a personal level, files are meant to be shared to those that you trust. Weak file permissions make it easy for the adversary to attack confidentiality and integrity. They can attack confidentiality as weak permissions allow them to access files they should not be able to access. Moreover, they can attack integrity as they might modify files that they should not be able to edit.

3. Access to Malicious Programs

Depending on the type of malicious program, it can compromise confidentiality, integrity, and availability. For instance, Trojan horses grant attackers access to your system, enabling them to read or modify files.

Certain malicious programs target availability. An example is ransomware, which encrypts user files, rendering them unreadable without the decryption password. Attackers offer to restore access by providing the encryption password if users pay the "ransom."

Network Security

A computer network is a group of computers and devices connected with each other. Network security focuses on protecting the security of these devices and the links connecting them. It consists of different hardware and software solutions to achieve the set of security goals. Hardware solutions refer to the devices you set up in your network to protect its security. They are hardware, so you can literally hold them with your hands. Examples of hardware appliance include:

  • Firewall appliance: The firewall allows and blocks connections based on predefined set of rules. It restricts what can enter and what can leave the network.
  • Intrusion Detection System (IDS) appliance: An IDS detects system and network intrusions and intrusion attempts. It tries to detect attackers' attempt to break into your network.
  • Intrusion Prevention System (IPS) appliance: An IPS blocks detected intrusions and intrusion attempts. It aims to prevent attackers from breaking into your network.
  • Virtual Private Network (VPN) concentrator appliance: A VPN ensures that the network traffic cannot be read nor altered by a third party. It protects the confidentiality (secrecy) and integrity of the data.

On the other hand, we have software security solutions. Common examples are:

  • Antivirus software: Antiviruses are meant to detect malicious files and block them from executing.
  • Host firewall: Unlike the firewall appliance, a hardware device, a host firewall is a program that ships as part of your system, or it is a program that you install on your system. For instance, MS Windows includes Windows Defender Firewall, and Apple macOS includes an application firewall; both are host firewalls.

Breaking into a target network usually includes a number of steps. According to Lockheed Martin, the Cyber Kill Chain has seven steps:

  • Recon: Recon, short for reconnaissance, refers to the step where the attacker tries to learn as much as possible about the target. Information such as the types of servers, operating system, IP addresses, names of users, and email addresses, can help the attack’s success.
  • Weaponization: This step refers to preparing a file with a malicious component, for example, to provide the attacker with remote access.
  • Delivery: elivery means delivering the “weaponized” file to the target via any feasible method, such as email or USB flash memory.
  • Exploitation: When the user opens the malicious file, their system executes the malicious component.
  • Installation: The previous step should install the malware on the target system.
  • Command and Control: The successful installation of the malware provides the attacker with a command and control ability over the target system.
  • Actions on Objectivs: After gaining control over one target system, the attacker has achieved their objectives. One example objective is Data Exfiltration (stealing target’s data).
